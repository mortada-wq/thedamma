# صاحب — Sahib Design System

> Single-file reference. Read this end-to-end before designing anything for Sahib.
> When tokens, asset paths, or rules below disagree with code in this repo,
> the code wins — but file an issue so we can sync.

---

## 1. The product in one paragraph

**Sahib (صاحب)** is an Arabic-native AI companion — a chat product whose
calligraphic word-mark and orange accent live on a single dark canvas. The voice
is warm and Khaleeji ("hello buddy", not "hello user"). The tagline
**أذكى من جن الماء** ("smarter than the water djinn") is folkloric, confident
and self-aware. Reach for the product's name (صاحب / Sahib) — never "the AI",
"the assistant", or "the model".

---

## 2. Voice & copy

- **Address the user as a friend.** The brand name is _Sahib_ — companion. The
  default placeholder is **سلم على صاحبك** ("say hi to your buddy"), not
  "Type a message".
- **Fragment, don't sentence.** Headlines and taglines are 2–6 word lines,
  Muna at tight `line-height: 100%`. The display copy wraps around the
  calligraphy, not the other way around.
- **No emoji. No exclamation marks** (except the deliberately
  exclamation-bearing tagline **مجنح!**). Calmness is part of the premium feel.
- **No filler.** Don't pad screens with placeholder text. Compose around the
  word-mark and prompt bar.

## 3. Color & Theme system

Sahib is designed around a highly resilient, twin-engine visual system. If the team disappears, any future developer must be able to restore Sahib's exact visual signature by following these structural laws.

### 3.1 Theme Philosophy
- **Dark Mode (Default - Canonical)**: Represents an immersive, late-night obsidian canvas. It utilizes low-glow, anti-glare surfaces designed for comfortable, prolonged companion interactions. It is high-tech, calligraphic, and premium.
- **Light Mode (Warm Matte Paper)**: Designed for daytime legibility and high visual accessibility. It avoids harsh stark whites in favor of a warm, print-like matte aesthetic (`#f8f9fa` canvas) with soft physical shadows and deep text contrast.

### 3.2 Theme Orchestration
The active theme is toggled by setting the `data-sahib-theme` attribute on the root `<html>` element:
```javascript
// Toggle logic
document.documentElement.dataset.sahibTheme = "dark" | "light";
```
This attribute switches the global custom CSS properties in `src/styles/sahib-themes.css` which dynamically cascade through the entire DOM.

### 3.4 Element-Specific Component Adaptations

To maintain structural clarity and prevent visual regression, these specific components must adapt dynamically under their respective selectors:

#### 1. The Dynamic Prompt Bar (`.sh-prompt--chat`)
- **Dark Mode**: 
  - Uses an animated glowing border sweep (`#F7731E` to `#5E94FF` rotating over 6s infinite via standard padding mask technique).
  - Background is volcanic dark glass (`#2b2b2b`).
  - Active `:focus-within` leverages the glowing rim for high-tech intent.
- **Light Mode**:
  - The animated glowing border is completely disabled (`display: none !important`) to eliminate screen glare and maintain a print-like visual calm.
  - Border is a crisp solid hairline (`rgba(0,0,0,0.08)`).
  - Background is pure paper white (`#ffffff`) with subtle physical shadows (`box-shadow: 0 10px 30px rgba(0,0,0,0.04)`).
  - Active `:focus-within` instantly changes the border to solid `--accent-orange` (`#F7731E`) and applies a warm, premium focus glow:
    ```css
    border-color: var(--accent-orange) !important;
    box-shadow: 0 10px 30px rgba(247, 115, 30, 0.08), 0 0 0 1px var(--accent-orange) !important;
    ```

#### 2. The Sidebar Navigation (`.sh-sidebar-shell nav`)
- **Dark Mode**:
  - Translucent obsidian body (`rgba(30, 30, 30, 0.96)`) with subtle white-highlight hovers (`rgba(255,255,255,0.06)`).
  - Item icons render in crisp `#E7E7E7`.
- **Light Mode**:
  - Warm grey solid body (`#f3f4f6`) with a clean border line (`rgba(0,0,0,0.05)`).
  - Highlights use a very light soft grey wash (`rgba(0,0,0,0.035)` on hover, `rgba(0,0,0,0.05)` active).
  - Item icons and labels transition dynamically to high-contrast `--text-secondary` (`#495057`) to preserve daylight legibility.

#### 3. Custom Scrollbars (`*::-webkit-scrollbar`)
- **Dark Mode**:
  - Standard dark-transparent scrollbars.
- **Light Mode**:
  - Sleek custom thin scrollbars (width/height `6px`) are injected globally:
    - Track: Transparent to stay completely hidden.
    - Thumb: Sleek grey-blue rounded pill (`rgba(90, 96, 112, 0.28)`), darkening to (`rgba(90, 96, 112, 0.45)`) on hover.

#### 4. Chat Bubbles
- **User Bubble**:
  - *Dark Mode*: Injected on `--surface-2` (`#2b2b2b`).
  - *Light Mode*: Switches to a solid pure white background (`var(--surface-1)`) with a very soft hairline border, ensuring the user's text stands out clean on the warm-grey canvas.
- **Sahib Bubble**:
  - Transparent fill in both modes, bordered by `--hairline` (`rgba(255,255,255,0.08)` in Dark, `rgba(0,0,0,0.06)` in Light), keeping Sahib's elegant Muna typography as the central focal point of the page.

## 4. Typography

**Muna** is the only family for chrome. **Aref Ruqaa** is added for one
purpose only: song titles in the music generation page.

Both fonts ship as `.ttf` (`fonts/Muna-Regular.ttf`, `fonts/Muna-Black.ttf`).
Aref Ruqaa loads from Google Fonts (`family=Aref+Ruqaa:wght@400;700`).

### Scale

| Token | px | Weight | Line-height | Use |
|---|---|---|---|---|
| `--fs-display` | 80 | Black | 100% | Calligraphic hero only (word-mark size). |
| `--fs-h1`      | 40 | Black | 100% | Section heroes. |
| `--fs-h2`      | 28 | Black | 120% | Section titles. |
| `--fs-h3`      | 22 | Regular | 130% | Sub-titles. |
| `--fs-body`    | 20 | Regular | 140% | Chat text, default body. |
| `--fs-meta`    | 16 | Regular | 140% | Metadata, timestamps. |
| `--fs-caption` | 13 | Regular | 140% | Captions. |

### Aref Ruqaa — song titles only

```css
font-family: var(--font-song);   /* "Aref Ruqaa", "Muna", serif */
font-weight: 700;                /* or 400 for ribbon-style alternates */
font-size: 32px;
line-height: 1.25;
```

Use **only** for song titles in the music generation page. Do not use Aref
Ruqaa for body, headings, or marketing copy.

### Latin fallback

Stack: `"Muna", "Tajawal", "IBM Plex Sans Arabic", ui-sans-serif, system-ui`.
Tajawal is an unapproved substitution; flag it whenever it appears.

## 6. Layout & RTL

- **`direction: rtl` is global.** Sahib is Arabic-first.
- **Sidebar lives on the visual right.** Implementation note: under `dir="rtl"`
  + `display: flex` + `flex-direction: row`, the **first** flex child renders
  on the visual right. Source order `<Sidebar/> … <main/>` is what we use.
- **Chat-canvas radius is asymmetric:** `border-radius: 27px 14px 14px 14px`.
  The 27 sits on the top-right (visual reading start in RTL).
- **Cards default to radius 14**; the prompt bar uses **radius 17**;
  pills use **999**.

---

## 7. Borders, shadows, radii

- **Hairlines:** `rgba(255,255,255,0.06)`, 0.4–1px.
- **Orange strokes:** 1.5–2px. Used to declare interactive intent.
- **Blue strokes:** 1px solid `#5E94FF`. Default for non-CTA buttons and chips.
- **Shadows:** short, downward, low opacity. No glows.

```
--shadow-sm:   0 1px 1px 0 rgba(0,0,0,0.25);
--shadow-md:   1px 1px 1px 0 rgba(0,0,0,0.25);
--shadow-logo: 0 1px 2px 0 rgba(0,0,0,0.32);
--shadow-glow: 0 1px 1px 0 rgba(0,0,0,0.59);
```

Radii scale: `6 / 10 / 14 / 17 / 27`.

---

## 8. Motion

- **No bouncy easing.** Use `ease` or `ease-out`, 120–200ms.
- **Hovers** fade colour with a 6–10% wash in the stroke color. Never scale.
- **Press states** scale to `0.98` only.
- The **sidebar toggle slides horizontally**; expanded labels fade in after the
  slide completes.
- The **prompt-bar gradient rim** sweeps left→right over 6s linear infinite.
- Word-mark appears as a fade-in. No parallax, no rotate.

## 9. Components

### 9.1 Sidebar (collapsed default)

- 51px wide collapsed; expands to ~240px on toggle.
- Sits at the visual right (RTL).
- Top hairline gradient: cool blue fading to canvas.
- **Toggle glyph at the top** is `assets/sidebar-toggle.svg` — a five-petal
  star in muted blue gradient (`#395387 → #75A3FF → #385387`).
- Item icons are 18–22px, solid fills, color `#E7E7E7`.

**Order (top → bottom):**

1. **محادثة** (new chat) — circle with `+`. Inline shape; no SVG.
2. **المحفوظات** (library — chats, songs, image generations, archive) — `assets/icon-library.svg`. Keeps its native white→blue gradient (the only icon that does).
3. **عميل** (agent) — `assets/icon-agent.svg`.
4. **صوت** (voice) — `assets/icon-voice.svg`.
5. **صورة** (image generation) — `assets/icon-image.svg`. Thin photo/landscape glyph.
6. **موسيقى** (music) — `assets/icon-music.svg`.

The old `assets/icon-music-inner.svg` is **deprecated** — use `icon-music.svg`.

### 9.2 Dynamic Expandable Prompt bar

The Prompt Bar is designed around a **collapsible, multi-state container** that scales perfectly across devices without disrupting layout flow.

- **Standard Dimensions**: 728px max width × 56px base height (up to 250px max auto-expand height during multi-line input), radius **17**.
- **The Collapsed State (Default)**:
  - Cleanest possible canvas. To avoid clutter, **only the Orange Send Button** is visible inside the main bar area.
  - Textarea is right-aligned (`dir="rtl"`), starting with the default placeholder: **`سلم على صاحبك`**.
  - On the bottom-right side, the active **Agility Mode** (e.g., **عجل** with its secondary tagline *"وَمن الحُلم أن تكون عجولا"*) is shown.
  - A small, minimal **downward arrow chevron ($\vee$)** sits centered on the bottom border.
- **The Extended State (Active Expansion)**:
  - When the user clicks the downward chevron, the container's height expands downwards.
  - A subtle divider line separates the text area from the new bottom compartment.
  - The bottom compartment reveals a horizontal compartment holding **exactly 9 simple utility icons** spaced symmetrically.
  - The center trigger chevron rotates 180° to point **straight up ($\wedge$)** to allow collapsing.
  - Designed to be fully responsive, the extended card scales beautifully on mobile viewports.

### 9.3 Send button

The **one solid orange element** in the system, anchoring the eye to the key primary action.

- **RTL Alignment**: Always positioned on the **visual far-left** of the input bar (opposite to the Arabic reading start point).
- **Proportions**: 32px × 32px (compact toolbar) or 36px × 36px (chat main).
- **Style**: Fully circular (border-radius 999), `#F7731E` solid fill.
- **Icon**: A sharp, bold **white up-arrow ($\uparrow$)** centered in the circle (drawn at stroke-width 2.4).
- **States**:
  - Hover: Subtle scale transition up to `1.05` and background shift to `#D46402`.
  - Press: `scale(0.96)` for tactile physical click feedback.
  - Disabled: `opacity: 0.4`, cursor `default`, returns to base scale.

### 9.4 Stroke buttons

Transparent fill, 1.5px stroke, pill radius (999). Four stroke colors:

| Stroke | Color | Use |
|---|---|---|
| **blue** | `#5E94FF` | Default — everyday CTA, secondary actions. Most buttons. |
| **grey** | `#A9A9A9` | Tertiary — cancel, dismiss. |
| **white** | `#FFFFFF` | Plain, prominent neutral. |
| **orange** | `#F7731E` | Attention. **Rare** — reserve for one high-intent action per surface (send, submit). |

Hover: stroke-color wash at 6–10% opacity. Press: `scale(0.98)`. AR padding
defaults to `7px / 14px`; LTR overrides to `10px / 20px`.

### 9.5 Suggestion chips

Sit **under the prompt bar** (not above the hero).

- Thin **1px solid `#5E94FF`** stroke (no gradient).
- Pill (`border-radius: 999`).
- Text in **Muna Regular** (not bold), color `#5E94FF`.
- Padding `7px 14px`.
- Hover: `rgba(94,148,255,0.10)` background wash. Press: `scale(0.97)`.

### 9.6 Chat bubbles

- **User bubble:** `--surface-2` fill, `border-radius: 14 14 4 14` (tail toward
  the message author).
- **Sahib bubble:** transparent + 1px hairline border, `border-radius: 14 14 14 4`.
- Text: Muna 18px, 1.45 line-height.
- Timestamps below in `--text-muted`, format `H:MM ص/م · أنت / صاحب`.

Sample copy:

> User: سلام صاحب. اشلونك اليوم؟
> Sahib: هلا والله. حلو كلش يخبل، اكلك اليوم أمي شنو طابخة؟

### 9.7 Agility modes (4 horses)

Four modes representing thinking pace, surfaced through the prompt bar's
agility button. **عجل is the default.**

| # | Asset | Label | Vibe |
|---|---|---|---|
| 1 | `assets/agility-1-gallop.png` | **عجل** (ajil) | Fast · default. Galloping horse. |
| 2 | `assets/agility-2-walk.png` | **مهل** (mahl) | Calm · measured. Walking horse. |
| 3 | `assets/agility-3-knight.png` | **فطن** (fatin) | Sharp · strategic. Chess knight. |
| 4 | `assets/agility-4-pegasus.png` | **مجنح!** (mujannah!) | Creative · free. Winged pegasus. |

Visual treatment in the picker:

- 4-up grid, each tile 14px-radius card on `--canvas-bg`, hairline border.
- Active tile: 1px `#5E94FF` border + 12%-blue gradient wash at the top.
- Horse images: `filter: brightness(0) invert(1)` to render pure white.
- Labels under each horse in Muna 17, sub-labels in Muna 12 muted.

In the prompt bar, the **active horse** displays at 18px height in the right
cluster's agility slot. When the agility mode is conceptually "off" (rarely
shown), drop the image opacity to 0.5.

### 9.8 Welcome hero

Two states:

- **First-time sign-in** — full calligraphic word-mark + the tagline beneath
  it. Use `<WordMark width={280} />` (stacked `logo-back.svg` + `logo-front.svg`).
- **Returning user (default)** — small five-petal blue star + a conversational
  Arabic question to the right of it. Implemented as a single SVG asset
  `assets/welcome-returning.svg` that packs both glyphs. Default greeting:
  **شلونك اليوم؟** ("how are you today?"). Sits directly above the prompt bar.

Both states use `<img>` — never try to mask-image SVGs at runtime, `mask-mode`
defaults will break the gradients.

Below the welcome hero (either state) sit the 4 blue-stroke suggestion chips
(§9.5).

### 9.9 Launch Progress Bar

A special component mounted on the Welcome Page (`!hasStarted` view) to visualize launch milestone progress for the product.

- **Theme-Aware Styling**: Built with CSS variable foundations. Hardcoded white backgrounds/borders/text styles are strictly forbidden to ensure perfect legibility under both themes:
  - Wrapper background must use `var(--surface-1)` with border `var(--surface-edge)` (solid `#262626` in dark mode, `#dee2e6` in light mode).
  - Main text label uses `var(--text-secondary)`, and sub-labels ("من ٩٠٪") use `var(--text-muted)`.
  - Progress bar track background uses `var(--surface-2)` with border `var(--hairline)`.
  - Active track utilizes a gorgeous `bg-gradient-to-l from-[#F7731E] to-[#5E94FF]`.
- **No Glows / Shadow Rule**: Complying with Section 7 shadow laws, all glows (such as track glows) are strictly disabled (`shadow-none`), and the wrapper card utilizes a flat standard `shadow-sm` or `var(--shadow-sm)`.
- **Capped Calibration**: Shows current milestone progress up to a maximum threshold of **٩٠٪** (90%) as configured in the administrative calibration model.
- **Admin Calibration Hub**: Restricted to admins (`isAdmin === true` in sidebar context). Clicking on the link displays the `LaunchProgressHub` modal to calibrate individual progress bars across 15 custom launch tasks. All progress details are dynamically persisted in standard local storage.
